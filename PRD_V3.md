## 1. 제품 개요

### 한 줄 정의

AI로 만든 인터랙티브 앱을 올리고, 외부 SNS에서 바로 실행 가능한 앱 체험으로 연결하며, 유저는 피드에서 직접 체험·리믹스하고, 창작자는 체험 기반 수익과 데이터를 얻는 플랫폼.

### MVP 슬로건

> **"올리고 → 바로 놀아보는"**

창작자는 이미 Claude Code, Cursor, Replit, Vercel, GitHub Pages 등으로 앱을 만든다. nuevo의 MVP는 앱을 만드는 도구가 아니라, **이미 만든 앱을 올리고, 공유하고, 발견되게 하고, 직접 체험시키는 배포/발견 플랫폼**이다.

### V3 핵심 포지셔닝

v0.5까지의 PRD는 "앱을 올리는 피드"에 가까웠다. 고객 인터뷰 결과, 채택 조건은 더 명확하다.

1. **외부 플랫폼 브릿지**: Instagram, TikTok, YouTube, Reddit에서 nuevo 앱 링크를 누르면 3초 안에 직접 체험으로 진입해야 한다.
2. **발견성**: 시간순 탐색 탭만으로는 부족하다. MVP부터 태그 검색과 리믹스 노출 증폭이 필요하다.
3. **리믹스 바이럴**: 리믹스는 단순 수익 공유가 아니라 원본을 재발견시키는 성장 루프여야 한다.
4. **수익 신뢰성**: WES 점수와 정산 풀, raw 이벤트 로그가 창작자에게 검증 가능해야 한다.
5. **창작자 소유권**: 앱 메타데이터, 팬 관계, WES 데이터는 창작자가 export할 수 있어야 한다.

### MVP 핵심 루프

```text
[창작자]
가입 → URL 제출 → 호환성 진단 → 미리보기 → 태그/외부 링크 입력 → 게시
→ 탐색 탭 노출 확인 → SNS 딥링크 공유

[체험자]
SNS 링크/탐색/검색 → 앱 즉시 실행 → 반응/댓글 → 리믹스 → 내 버전 게시

[원본 창작자]
첫 체험 알림 → 리믹스 알림 → 리믹스 노출 증폭 → WES/이벤트 로그 확인
```

### 포지셔닝

| 플랫폼 | nuevo가 벤치마킹한 것 | nuevo가 다른 것 |
|--------|----------------------|----------------|
| Instagram | 피드 UX, 팔로우, 외부 트래픽 유입 | 영상에서 끝나지 않고 앱을 직접 실행 |
| TikTok | 리믹스/바이럴 루프 | Duet/Stitch식 리믹스를 앱 체험으로 확장 |
| YouTube | 검색 기반 롱테일, 창작자 수익 배분 | 튜토리얼 영상과 완성 앱을 양방향 연결 |
| Reddit / itch.io / CodePen | 커뮤니티 쇼케이스, 창작자 신뢰 | AI 앱 특화 + 체험 기반 WES + 수익/데이터 투명성 |

## 2. 타겟 사용자

### 창작자 (Creators)

- **주요 타겟**: Claude Code, Cursor, Replit로 앱을 만드는 코딩 비전공자 / 바이브코더
- **행동 패턴**: 만든 앱을 영상으로 찍어 Instagram Reels, TikTok, YouTube Shorts/튜토리얼에 올림
- **핵심 니즈**: "영상 말고 앱 자체를 공유하고 싶다. 사람들이 바로 실행해봤으면 좋겠다."
- **채택 조건**: 외부 SNS 딥링크, 태그 검색, 리믹스 노출, 수익/데이터 투명성

### 체험자 (Users)

- **주요 타겟**: AI 창작물에 관심 있는 20-35세, 새로운 인터랙티브 경험을 즐기는 사람
- **행동 패턴**: SNS에서 AI 앱 시연을 보고 "나도 해보고 싶다"고 느낌
- **핵심 니즈**: 가입 없이 먼저 실행해보고, 재미있으면 반응/댓글/리믹스하고 싶음

### 브랜드 (Advertisers / Sponsors)

- AI 네이티브 콘텐츠와 인터랙티브 광고에 관심 있는 브랜드
- 일반 영상 조회보다 "직접 조작한 시간"과 "반응/리믹스" 데이터를 보고 싶어함
- MVP에서는 내부 운영팀이 수동 매칭하고, Phase 2에서 셀프서브 대시보드로 확장

## 3. MVP 핵심 기능

### 3.0 인증 & 비로그인 체험

| 기능 | 설명 | MVP |
|------|------|-----|
| Google 소셜 로그인 | 가입/로그인 원클릭 | ✅ |
| 프로필 설정 플로우 | 첫 가입 후 handle, 이름, 아바타 설정 | ✅ |
| 인증 상태 전역 관리 | 현재 유저 정보와 보호 라우트 관리 | ✅ |

#### 비로그인 허용 범위

| 기능 | 비로그인 허용 여부 | 비고 |
|------|------------------|------|
| 피드 탐색 | ✅ 허용 | 신규 유입 전환율 우선 |
| 태그 검색 | ✅ 허용 | 앱 발견성 우선 |
| 앱 체험 | ✅ 허용 | iframe 실행 가능 |
| 반응 | ❌ 로그인 필요 | 중복 방지 |
| 댓글 | ❌ 로그인 필요 | 커뮤니티 품질 관리 |

| 업로드 | ❌ 로그인 필요 | 창작자 계정 필요 |

| 탭 | 정렬 기준 |

| 홈 피드 | 생성시간 역순 + 팔로우 창작자 우선 |

#### 외부 링크 필드

창작자는 앱 페이지에 최대 3개의 외부 링크를 추가할 수 있다.

예시:
- "이 앱의 제작 과정 보기" → YouTube
- "GitHub 보기"
- "원본 릴스 보기"
- "Reddit 토론 보기"

`posts.external_links`는 JSON 배열로 저장한다.

리믹스 기능의 목적은 두 가지다.

1. 원본 창작자에게 파생 수익을 제공한다.
2. 리믹스가 원본을 다시 발견시키는 바이럴 노출 루프가 된다.

> **결정**: MVP에서는 리믹스 허용을 기본값으로 유지한다. 작품별 리믹스 허용/비허용 선택권은 초기 100명 창작자 인터뷰 후 결정한다.

### 3.5 멀티 플랫폼 공유 딥링크

창작자가 외부 SNS에서 nuevo 앱을 공유할 때, 링크 클릭 즉시 앱 체험 화면으로 진입하는 딥링크를 제공한다.

#### 공유 URL 구조

```text
기본 URL: https://nuevo.app/@{handle}/{app-slug}
딥링크:   https://nuevo.app/@{handle}/{app-slug}?autoplay=true
```

`autoplay=true`는 앱 페이지 진입 시 iframe을 즉시 전개한다. 사용자는 별도 탭 없이 앱 체험 화면에 들어온다.

#### 플랫폼별 최적화

| 채널 | 요건 | MVP |
|------|------|-----|
| Instagram 바이오 / 릴스 링크 | OG title/image/description 최적화, 인앱 브라우저에서 즉시 실행 | ✅ |
| TikTok 인앱 브라우저 | iframe/sandbox 호환성 테스트, `autoplay=true` 지원 | ✅ |
| YouTube 설명란 | 클릭 후 앱 체험 자동 진입, 튜토리얼 역방향 링크 지원 | ✅ |
| Reddit / Discord / 카카오톡 | 링크 미리보기 이미지와 설명 표시 | ✅ |

#### 공유 기능

- 앱 페이지 내 "공유하기" 버튼
- 딥링크 클립보드 복사
- Instagram/TikTok/YouTube/Reddit용 공유 문구 템플릿
- OG 이미지는 앱 미리보기 스크린샷을 우선 사용
- 모바일 기준 앱 체험 진입 LCP 목표: 3초 이하

## 4. 앱 임베딩 스펙

창작자가 제출하는 앱은 아래 스펙을 충족해야 한다.

```text
뷰포트: 390px iPhone 기준 최적화 권장
높이: 자유, 세로 스크롤 허용
공개 접근: 인증 없이 접근 가능한 URL
iframe 허용: X-Frame-Options DENY/SAMEORIGIN 금지
권장 플랫폼: Vercel, GitHub Pages, Glitch, CodePen, Replit
카메라/마이크: 사용 시 contentType 필드에 선언
```

### 4.1 sandbox / 격리

기존 v0.4의 `allow-same-origin + allow-scripts` 조합은 sandbox 우회 위험이 있으므로 제거한다.

```html
<iframe
  sandbox="allow-scripts allow-forms allow-popups"
  allow="camera; microphone"
/>
```

창작자 앱은 가능하면 `{creator-handle}.nuevo.app` 또는 격리된 preview origin에서 서빙한다. same-origin 권한을 주지 않고도 실행성과 보안을 동시에 확보하는 것이 목표다.

### 4.2 iframe 진단 도구

URL 제출 전 "내 앱이 nuevo에서 제대로 뜰까?"를 확인한다.

1. 창작자가 URL 입력
2. 서버사이드 `POST /api/check-url`에서 해당 URL의 HTTP 헤더 fetch
3. 아래 항목 체크 후 결과 표시

| 항목 | 체크 기준 | 결과 |
|------|-----------|------|
| X-Frame-Options | DENY/SAMEORIGIN이면 경고 | 임베딩 차단 |
| CSP `frame-ancestors` | `'none'` 또는 nuevo 미허용이면 경고 | CSP 차단 |
| HTTPS | HTTP이면 경고 | 혼합 콘텐츠 주의 |
| 공개 접근 | 200 응답이면 통과 | 접근 가능 |

런타임 이슈, 카메라/마이크 권한, 모바일 스크롤 동작은 미리보기 iframe에서 창작자가 직접 확인한다.

### 4.3 WES SDK / postMessage 프로토콜

cross-origin iframe에서는 내부 클릭, 터치, 키 입력을 직접 수집할 수 없다. 체험 시간과 인터랙션 기반 WES 정밀 측정을 위해 선택형 SDK를 제공한다.

```html
<script src="https://sdk.nuevo.app/wes.js"></script>
```

SDK 이벤트:

| 이벤트 | 설명 |
|--------|------|
| `session_start` | 앱 최초 로드 |
| `interaction` | 클릭, 터치, 키 입력 |
| `session_end` | 탭 닫힘 또는 포커스 이탈 30초 이상 |

직접 구현 시:

```javascript
window.parent.postMessage({
  type: 'nuevo_event',
  event: 'interaction',
  timestamp: Date.now()
}, 'https://nuevo.app');
```

SDK 미적용 앱의 처리:

- WES의 "총 체험 시간" 항목은 0으로 처리
- iframe onLoad 기반 체험 세션 수, 반응, 댓글, 리믹스는 정상 집계
- 업로드 시 "SDK 미적용 앱은 체험 시간 기반 WES 점수를 받을 수 없습니다"라고 고지

## 5. 수익 모델 / WES

### 5.1 수익 구조

```text
플랫폼 총 수익
├── 광고 수익
│   ├── 체험 전 프리롤 광고
│   └── 체험 중 비침투적 배너
├── nuevo+ 구독 수익
│   ├── 광고 제거
│   └── 창작자 분석 도구
└── 브랜드 스폰서십
    └── 브랜드가 창작자에게 인터랙티브 앱 제작 의뢰
```

> 광고 수익 60%, 구독 수익 30%, 브랜드 스폰서십 10% 같은 비율은 MVP에서 확정값이 아니라 가설이다. V3에서는 비율보다 **정산 풀의 출처와 공개 방식**을 더 중요한 요구사항으로 둔다.

### 5.2 WES 계산

WES는 조회수가 아니라 직접 체험과 참여를 반영한다.

```text
WES = (유효 체험 세션 수 × 1.0)
    + (총 체험 시간(분) × 0.8)
    + (반응 수 × 1.5)
    + (댓글 수 × 2.0)
    + (리믹스 수 × 5.0)
```

단, 총 체험 시간은 SDK 또는 직접 postMessage 이벤트를 적용한 앱만 집계한다.

### 5.3 서버사이드 검증

수익과 연결되는 이벤트는 클라이언트 값을 그대로 신뢰하지 않는다.

| 검증 항목 | MVP 구현 |
|-----------|----------|
| 세션 유효성 | 동일 IP/디바이스에서 60초 내 10회 이상 세션 시작 시 무효 |
| 비정상 체험 시간 | 단일 세션 2시간 초과 시 상한 적용 |
| 반응 중복 | 동일 유저-포스트 조합 서버사이드 upsert |
| 봇 플래그 | User-Agent 없음, 비정상 SDK 이벤트 패턴 탐지 |
| 비로그인 이벤트 | 플랫폼 지표에는 포함, 정산 WES에는 기본 제외 |

Phase 3에서 이상 탐지와 정산 검증을 고도화한다.

### 5.4 정산 풀 투명 공개

WES가 창작자에게 신뢰받으려면 "내 점수가 어떻게 돈으로 바뀌는지"를 계산할 수 있어야 한다.

#### MVP 창작자 대시보드 공개 항목

| 항목 | 공개 방식 |
|------|-----------|
| 나의 월별 WES | 세션, 시간, 반응, 댓글, 리믹스 항목별 분해 |
| WES raw 이벤트 로그 | CSV export |
| 예상 수익 | 정산 풀 확보 전까지 "예상"으로 표기 |
| 확정 수익 | 정산 월 마감 후 확정 금액 표기 |
| 플랫폼 총 WES 풀 | 전체 WES 총점과 내 비율 표시 |
| 정산 풀 출처 | 광고 / 구독 / 브랜드 스폰서십 구분 |

### 5.5 수익 시뮬레이션 예시

아래 숫자는 약속이 아니라 MVP 가설 설명용 예시다.

| 시나리오 | 월 체험 세션 | 예상 광고 수익 | 창작자 배분 예시 | 해석 |
|----------|--------------|----------------|------------------|------|
| 새싹 진입 | 500 | 약 500원 | 약 300원 | 수익보다 첫 체험 확보가 중요 |
| 성장 진입 | 5,000 | 약 5,000원 | 약 3,500원 | 부수 수익 수준 |
| 프로 진입 | 50,000 | 약 50,000원 | 약 37,500원 | 의미 있는 월 수익 시작 |
| 챔피언 | 500,000 | 약 500,000원 | 약 400,000원 | 상위 창작자용 |

MVP 초기의 주 동기는 "돈을 많이 번다"가 아니라 "내 앱을 직접 체험시킬 수 있다"이다. 수익화 메시지는 과장하지 않고, WES 데이터와 브랜드 스폰서십으로 점진 검증한다.

### 5.6 리믹스 수익 공유

- 1단계 리믹스: 원본 창작자에게 리믹스 수익의 10% 귀속
- 2단계 리믹스: 원본 창작자 5%, 직전 창작자 5%
- MVP에서는 리믹스 depth 2단계까지만 정산 대상으로 명시
- 3단계 이상, 탈퇴 계정, 저작권 분쟁은 Phase 3 정산 구현 전 법무 검토

### 5.7 브랜드 스폰서십

브랜드가 특정 창작자에게 인터랙티브 앱 제작을 의뢰한다.

MVP:
- nuevo 운영팀이 수동 매칭
- 창작자는 "스폰서 체험" 앱을 제작해 업로드
- 브랜드는 체험 수, 평균 체험 시간, 반응, 리믹스 수 리포트를 받음
- nuevo는 중개 수수료를 받음

Phase 2:
- 창작자 공개 WES 프로필
- 브랜드 협업 제안 인박스
- 스폰서 체험 성과 리포트 공유 링크
- 셀프서브 브랜드 캠페인 대시보드

## 6. 크리에이터 파트너십 / 데이터 소유권

### 6.1 파트너십 프로그램

#### 가입 조건

1. 계정 생성 30일 이상
2. 작품 최소 3개 이상 게시
3. 최근 30일 유효 체험 세션 기준 충족
4. 커뮤니티 가이드라인 위반 없음

> 기존 "월 500 체험 세션" 기준은 근거 데이터가 부족하다. MVP 초기 100명 창작자 데이터 수집 후 재조정한다. 출시 초기 얼리 파트너는 한시적으로 100 세션 기준을 테스트할 수 있다.

#### 혜택

- 수익 배분 참여
- 크리에이터 스튜디오
- WES raw event log CSV export
- 프로필 파트너 배지
- 브랜드 스폰서십 매칭 우선권
- 신기능 얼리 액세스

### 6.2 창작자 데이터 소유권

nuevo는 창작자의 콘텐츠와 데이터에 대한 소유권을 다음처럼 보장한다.

| 데이터 유형 | 소유권 | export 방법 |
|-------------|--------|-------------|
| 올린 앱 URL + 메타데이터 | 창작자 소유 | JSON export |
| 팬 관계 | 창작자 소유 | CSV export |
| WES 이벤트 로그 | 창작자 소유 | 월별 CSV export |
| 댓글 / 반응 데이터 | 창작자 소유 | JSON export |
| 외부 링크 / 태그 | 창작자 소유 | JSON export |

정책:
- 계정 탈퇴 전 데이터 export 제공
- 탈퇴 후 30일 유예 기간
- 서비스 종료 시 90일 전 고지 및 전체 데이터 export 제공
- 창작자 데이터를 제3자에게 무단 판매하지 않음
